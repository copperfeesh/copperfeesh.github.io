Static Sites and Performance

The modern web is heavy. The average website today is several megabytes, filled with JavaScript bundles, tracking scripts, and oversized assets. But for a personal blog, do we really need all that?

Returning to static HTML and CSS isn't just a nostalgic choice—it's a performance one. By eliminating client-side rendering and database lookups, we can achieve near-instant load times and a significantly lower carbon footprint.

This blog is built to be fast, accessible, and resilient. It doesn't need a server to think before it serves; it's just files on a disk, ready for you to read.
